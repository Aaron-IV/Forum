# Real-Time Forum

Форум с поддержкой чата в реальном времени через WebSocket.

## Требования

- **Go** 1.26+ ([скачать](https://go.dev/dl/))
- **GCC** (нужен для сборки SQLite — `go-sqlite3` использует CGO)
  - macOS: установите Xcode Command Line Tools → `xcode-select --install`
  - Linux: `sudo apt install gcc` (или аналог для вашего дистрибутива)

## Запуск

### Вариант 1 — через `go run` (для разработки)

```bash
cd Forum
go run .
```

### Вариант 2 — сборка и запуск

```bash
cd Forum
go build -o forum .
./forum
```

> **Важно:** запускать нужно именно из корневой папки проекта (`Forum/`), потому что сервер ищет папку `static/` и файл базы данных `forum.db` относительно текущей директории.

После запуска откройте браузер и перейдите по адресу:

```
http://localhost:8080
```

## Частые ошибки

### Вижу список файлов/папок вместо сайта

Это значит, что вы открыли папку проекта напрямую в браузере (через `file:///...`). Так работать не будет — нужно сначала запустить сервер командой выше, а затем открыть `http://localhost:8080`.

### `cgo: C compiler not found`

Установите GCC (см. раздел «Требования»).

### Порт 8080 уже занят

Другой процесс уже использует порт 8080. Найдите и остановите его:

```bash
lsof -i :8080
kill <PID>
```

## Структура проекта

```
Forum/
├── main.go            # Точка входа
├── server/            # Настройка HTTP-сервера и маршрутов
├── handlers/          # Обработчики API (auth, posts, comments, messages)
├── database/          # Работа с SQLite (миграции, запросы)
├── middleware/        # Middleware (авторизация)
├── websocket/         # WebSocket хаб и обработчики
├── models/            # Модели данных
├── static/            # Фронтенд (HTML, CSS, JS)
│   ├── index.html
│   ├── css/
│   └── js/
├── go.mod
└── go.sum
```
